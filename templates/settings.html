Slack
Starts with xoxp-. Required to read DMs, private channels, and download attachments as you.
How to get this token
  1. Go to api.slack.com/apps and open your app (or create one)
  2. In the left sidebar click OAuth & Permissions
  3. Scroll to User Token Scopes and add: channels:history, groups:history, im:history, mpim:history, files:read, users:read
  4. Scroll up and click Install to Workspace → Allow
  5. Copy the User OAuth Token (starts with xoxp-)
Starts with xoxb-. Only needed as a fallback for public channels. The User Token above is preferred.
How to get this token
  1. Same app on api.slack.com/apps → OAuth & Permissions
  2. Scroll to Bot Token Scopes and add: channels:history, files:read, users:read
  3. Reinstall the app to your workspace
  4. Copy the Bot User OAuth Token (starts with xoxb-)
  5. Invite the bot to any channel: /invite @YourBotName
OpenAI
Used to analyze Slack threads and generate ticket drafts with GPT-4o (supports vision for images).
How to get this key
  1. Go to platform.openai.com/api-keys
  2. Click Create new secret key
  3. Give it a name (e.g. "Jira Ticket Creator") and click Create
  4. Copy the key immediately — it won't be shown again
  5. Make sure your account has a paid plan or credits — GPT-4o requires billing
Jira
Before you start: You need an Atlassian account with permission to create issues in your Jira project.
How to find your Jira details
  1. Server URL — Your Atlassian domain, e.g. https://yourcompany.atlassian.net. You can see it in your browser when logged into Jira.
  2. Project Key — The short prefix on your ticket numbers, e.g. if tickets are NET-123 the key is NET. Find it in Jira → Projects → your project → Project Settings.
  3. Account Email — The email you use to log into Atlassian.
  4. API Token — See below.

Your Atlassian workspace URL. No trailing slash.

The prefix on your ticket numbers, e.g. NET for NET-123.

The email address you use to log into Atlassian/Jira.

Not your password — a separate API token.
How to generate an API token
  1. Go to id.atlassian.com → Security → API tokens
  2. Click Create API token
  3. Give it a label (e.g. "Jira Ticket Creator") and click Create
  4. Copy the token — it won't be shown again after closing the dialog
  5. Note: the token inherits your account's Jira permissions — no extra scopes needed
